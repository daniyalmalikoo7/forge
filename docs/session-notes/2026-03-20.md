## Bugs fixed

### 1. RLS blocking server-side inserts (silent failure)
**Symptom:** POST /api/projects returned 201 but no row appeared in Supabase.
**Root cause:** The original `getClient()` could fall through to
SUPABASE_ANON_KEY. The anon key is subject to RLS policies, and without
an authenticated user auth.uid() returns null, so the policy
`owner_id = auth.uid()::text` silently blocked every insert (Supabase
returns 0 rows affected, not an error).
**Fix:** Split into two explicit clients — `adminClient()` using
SUPABASE_SERVICE_ROLE_KEY (bypasses RLS, for all server-side operations)
and `anonClient()` using SUPABASE_ANON_KEY (for future browser use).
All API routes now use adminClient().

### 2. .env not loaded when running from apps/api
**Symptom:** Even with correct .env at project root, env vars were
undefined when running `bun dev` from apps/api.
**Root cause:** Bun auto-loads .env from the current working directory
only. The dev script runs from apps/api/ but .env lives at the project root.
**Fix:** Updated apps/api package.json dev script to use
`--env-file=../../.env` so Bun explicitly loads the root .env.

## Things to watch for in Phase 7

- **Synthesis is the hardest agent.** It must produce a DiscoveryDocument
  that passes Zod validation. The prompt needs to be very explicit about
  the exact JSON structure required. Consider providing the Zod schema
  as part of the prompt, or at least a complete example document.
- **Token limits.** By the time Synthesis runs, the accumulated state
  (clarification + decomposition + research + requirements) is large.
  Monitor token usage — may need claude-opus-4-5 for Synthesis if
  claude-sonnet-4-5 truncates or loses context.
- **State field names.** We renamed `research` → `research_result` and
  `requirements` → `requirements_result` in ExplorerState to avoid
  LangGraph node/channel name collision. The new nodes must write to
  these renamed fields.
- **Prompt templates.** Each new agent needs a prompt in
  packages/prompts/src/explorer/ following the same PromptTemplate pattern.
  Don't forget to export from packages/prompts/src/explorer/index.ts and
  packages/prompts/src/index.ts.
- **Lazy model instantiation.** Use the same `getModel()` lazy pattern
  from clarifier/decomposer to avoid import-time crashes without an
  API key.
